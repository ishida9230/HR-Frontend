# 申請一覧ページのコンポーネント化

## 概要
申請一覧ページのコードを再利用可能なコンポーネントに分割し、保守性と再利用性を向上させました。

## 変更内容

### 新規作成コンポーネント

#### 1. `src/components/ui/table.tsx`
テーブル関連の共通コンポーネントを集約：
- `TableContainer`: 横スクロール可能なテーブルラッパー
- `Table`: テーブル本体（固定レイアウト対応）
- `TableHeader`: テーブルヘッダー
- `TableHeaderRow`: テーブルヘッダー行
- `TableHeaderCell`: テーブルヘッダーセル
- `TableBody`: テーブルボディ
- `TableRow`: テーブル行（クリック可能）
- `TableCell`: テーブルセル

#### 2. `src/components/list/RequestListTable.tsx`
申請一覧テーブルコンポーネント（list配下専用）
- テーブルコンポーネントを使用して実装
- 申請データの表示と行クリック処理を担当

#### 3. `src/components/list/RequestListSearchForm.tsx`
申請一覧検索フォームコンポーネント（list配下専用）
- 従業員名、部署、支店、役職、申請ステータスの検索フィールド
- 複数選択対応のPopoverコンポーネント
- 検索・リセットボタン

#### 4. `src/components/ui/pagination.tsx`
ページネーションコンポーネント（既存）
- 再利用可能なページネーションUI

#### 5. `src/components/ui/empty-state.tsx`
空状態表示コンポーネント（既存）
- テーブル内の空状態メッセージ表示

### 変更ファイル

#### `src/app/requests/list/page.tsx`
- 検索フォームとテーブルをコンポーネント化
- 不要なインポートを削除
- コードの可読性向上

## メリット

1. **再利用性**: テーブルコンポーネントは他のページでも使用可能
2. **保守性**: コンポーネント単位で修正・テストが容易
3. **可読性**: ページコンポーネントが簡潔になり、理解しやすくなった
4. **一貫性**: 共通コンポーネントにより、UIの一貫性が保たれる
